# PGQT Post-PL/pgSQL Polish Plan

This plan outlines the next phase of development for PGQT, focusing on data integrity (Enums), session management (Settings), and catalog completeness (Comments & ACLs).

---

## Rules of Engagement (Strict Verification)

For **EVERY** task implemented below, the following four-step process must be followed before proceeding to the next task:

1.  **Build Verification**: Run `cargo build --release` to ensure the project compiles.
2.  **Warning Cleanup**: Fix all compiler warnings (unused imports, variables, etc.).
3.  **Regression Testing**: Run `./run_tests.sh` and ensure all 320+ unit, integration, and E2E tests pass.
4.  **Documentation**:
    *   Create a new `.md` file in `docs/` if the feature is new.
    *   Update `CHANGELOG.md` and `docs/TODO-FEATURES.md`.
    *   Update `README.md` if appropriate.

---

## Phase 1: Enum Type Support (`CREATE TYPE ... AS ENUM`)

**Goal**: Support real PostgreSQL enums by transpiling them to SQLite `TEXT` columns with `CHECK` constraints.

### Tasks:
1.  **Catalog Update**:
    *   Implement `store_enum_value(type_name, label, sort_order)` in `src/catalog/mod.rs`.
    *   Ensure values are persisted in the `__pg_enum__` shadow table.
2.  **Transpiler Update**:
    *   Modify `reconstruct_create_enum_stmt` in `ddl.rs` to generate a `SELECT __pg_create_enum(...)` statement instead of a comment.
3.  **Constraint Injection**:
    *   In `reconstruct_column_def`, check if the `original_type` matches a known Enum in the catalog.
    *   If yes, append a `CHECK (col_name IN ('val1', 'val2'))` constraint to the SQLite column definition.
4.  **Runtime Integration**:
    *   Add `__pg_create_enum` internal function to `SqliteHandler`.

---

## Phase 2: Session Configuration (`set_config` / `SET`)

**Goal**: Enable persistent session-level settings that clients and ORMs often rely on.

### Tasks:
1.  **Session State**:
    *   Add `settings: HashMap<String, String>` to the `SessionContext` struct in `src/handler/mod.rs`.
2.  **Handler Logic**:
    *   Update the `set_config(name, value, is_local)` SQLite UDF.
    *   It must find the current `client_id` and update that session's `settings` map.
3.  **Setting Access**:
    *   Update the `current_setting(name)` SQLite UDF to read from the session's `settings` map before falling back to defaults.
4.  **Common Settings**:
    *   Ensure `timezone` and `application_name` are handled gracefully.

---

## Phase 3: Improved `LATERAL` Joins

**Goal**: Move from silent failure to explicit support/guidance for correlated subqueries.

### Tasks:
1.  **Subquery Detection**:
    *   Modify `reconstruct_range_subselect` in `src/transpiler/dml.rs` to detect `lateral: true`.
2.  **Validation**:
    *   If `lateral` is true and the operand is a subquery (not a function), add a transpilation error: "LATERAL subqueries are not yet supported in SQLite; try a window function or CTE."
3.  **Function Verification**:
    *   Ensure `LATERAL` continues to work (via omission) for table-valued functions like `jsonb_each`.

---

## Phase 4: `COMMENT ON` Storage Completion

**Goal**: Actually store the metadata generated by the `COMMENT ON` transpilation.

### Tasks:
1.  **OID Resolution**:
    *   Add a helper in `src/catalog/mod.rs` to resolve the `OID` of an object given its name and type (TABLE, COLUMN, INDEX, etc.).
2.  **Storage Logic**:
    *   Update `__pg_comment_on(type, name, comment)` in `src/handler/mod.rs` (currently a stub).
    *   It should resolve the OID and perform an `INSERT OR REPLACE` into the `__pg_description__` shadow table.
3.  **Catalog View Integration**:
    *   Ensure the `pg_description` and `pg_stat_all_tables` (if it exists) views show these comments.

---

## Phase 5: System Catalog Polish (ACLs & Types)

**Goal**: Achieve perfect compatibility with `psql` metadata commands (`\d`, `\df`, `\du`).

### Tasks:
1.  **ACL Mapping**:
    *   In `src/catalog/system_views.rs`, update `pg_class`, `pg_attribute`, and `pg_namespace`.
    *   Instead of `NULL` for `relacl`, `attacl`, etc., construct a PostgreSQL-compatible ACL string (e.g., `{postgres=arwdDxt/postgres}`) based on the `__pg_acl__` table.
2.  **Function Argument Types**:
    *   Update `pg_proc` view to correctly populate `proargtypes` using OIDs from `__pg_type__`.
3.  **Introspection E2E**:
    *   Add an E2E test that runs `psql -c "\d"` and `psql -c "\df"` and verifies no columns are reported missing by the client.

---

## Phase 6: Final Feature Freeze & Documentation

**Goal**: Consolidate all new features into the user-facing documentation.

### Tasks:
1.  **New Doc Files**:
    *   `docs/ENUMS.md`
