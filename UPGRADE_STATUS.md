### Root Cause of E2E Failures

The E2E tests each start/stop the proxy independently. When a function receives an unexpected type (e.g., Integer instead of String), it returns an error which causes the proxy to crash. The "No such process" errors occur because the proxy crashed on a previous test and can't be killed.

### Breaking Change Explanation

**Rusqlite 0.38** introduced stricter type checking in the `ctx.get::<T>(index)` method. Previously, it would accept integers when a String was expected and convert them. Now it strictly enforces types.

**Solution**: Use `ctx.get_raw(index)` which returns `ValueRef`, then match on the actual type and convert to String manually. This handles:
- Integers (e.g., `array_append('{1,2}', 3)`)
- Reals (floating point numbers)
- Text (strings)
- Blobs (binary data)
- Null values
